计算机网络 数据链路层

2023-11-29 计算机网络 知识

数据链路层属于计算机网络的低层。数据链路层使用的信道主要有以下两种类型：

  • 点对点信道。这种信道使用一对一的点对点通信方式。
  • 广播信道。这种信道使用一对多的广播通信方式，必须使用专用的共享信道协议来协调这些主机的数据发送。

本章最重要的内容是：

  • 数据链路层的点对点通信和广播信道的特点，以及这两种信道所使用的协议（PPP协议以及CSMA/CD协议）的特点。
  • 数据链路层的三个基本问题：封装成帧、透明传输和差错检测。
  • 以太网MAC层的硬件地址。
  • 适配器、转发器、集线器、网桥、以太网交换机的作用以及使用场合。

数据链路层的几个共同问题

数据链路和帧

链路（link）就是从一个节点到相邻节点的一段物理链路。
数据链路（data link）除了必须有一条物理链路外，还必须有一些必要的通信协议来控制这些数据的传输。
数据链路层的任务是将网络层交下来的数据报组装成帧，在两个相邻结点间的链路上传送以帧（frame）为单位的数据。
59

三个基本问题

  • 封装成帧
    封装成帧就是在一段数据的前后分别添加首部和尾部，这样就构成了一个帧。首部和尾部的一个重要作用就是进行帧定界。帧的数据部分长度上限叫做最大传送单元MTU（Maximum Tranfer Unit）。

  • 透明传输
    透明表示某一个实际存在的事物看起来却好像不存在一样，不管从键盘上输入什么字符都可以放在这样的帧中传输过去。

  • 差错检测
    在数据链路层广泛使用循环冗余检验CRC（Cyclic Redundancy Check）。
    对于要传送的数据M=101001（k=6），在M后面添加n个0，用得到（k+n）位的数除以双方实现商定的（n+1）位的除数P，得到的余数为R（n位，比P少一位），其中进行的是异或运算。这个余数R就作为冗余码拼接在数据M的后面发送出去。这种为了进行检错而添加的冗余码常称为帧检验序列FCS（Frame Check Sequence）。CRC是一种检错方法，而FCS是添加在数据后面的冗余码，在检错方法上可以用CRC，也可以不用CRC。
    在数据链路层若仅仅使用循环冗余检验CRC差错检测技术，则只能做到对帧的无差错接受，即凡是接收端数据链路层接收的帧，我们都能以非常接近于1的概率认为这些帧在传输过程中没有产生差错。

点对点协议PPP

PPP协议的特点

  • 简单
  • 封装成帧
  • 透明性
  • 多种网络层协议
  • 多种类型链路
  • 差错检测
  • 检测连接状态
  • 最大传送单元
  • 网络层地址协商
  • 数据压缩协商

PPP协议的组成

  • 一个将IP数据报封装到串行链路的方法。即支持异步链路（无奇偶检验的8比特数据），也支持面向比特的同步链路。
  • 一个用来建立、配置和测试数据链路连接的链路控制协议LCP（Link Control Protocol）。
  • 一套网络控制协议NCP（Network Control Protocol）其中每一个协议支持不同的网络层协议。

PPP协议的帧格式

各字段的意义

  • 首部的第一个字段和尾部的第二的字段都是标志字段F（Flag），规定为0x7E，表示一个帧的开始或结束。因此，标志字段就是PPP帧的定界符
  • 首部中的地址字段A规定为0xFF
  • 控制字段C规定为0x03
  • 协议字段为2字节。0x0021表示IP数据报；0xC021表示PPP链路控制协议LCP的数据；0x8021表示网络层的控制数据
  • 信息字段不超过1500字节
  • 尾部的第一个字段是使用CRC的帧检验序列FCS，占2字节

字节填充

当PPP使用异步传输时，它把转义字符定义为0x7D，并使用字节填充：
（1）把信息字段中出现的每一个0x7E字节转变成为2字节的序列（0x7D，0x5E）
（2）把信息字段出现的一个0x7D的字节转变为2字节的序列（0x7D，0x5D）
由于在发射端进行了字节填充，在接收端进行相反的变换即可正确地恢复出原来地信息。

零比特填充

当PPP协议使用同步传输采用零比特填充的方法实现透明传输，只要发现有5个连续1，则立即填入一个0，在接收端进行相反的变换即可正确地恢复出原来地信息。

PPP协议的工作状态

  • 当用户拨号接入ISP后，就建立了一条从用户个人电脑到ISP的物理连接。
  • 个人用户电脑向ISP发送一系列的链路控制协议LCP分组（封装成多个PPP帧），以便建立LCP连接。
  • 这些分组及其响应选择了将要使用的一些PPP参数，并进行网络曾配置，网控制协议NCP给新接入的用户个人电脑分配一个临时的IP地址，使用户个人电脑成为互联网上的一个有IP地址的主机。
  • 当用户通信完毕时，NCP释放网络层连接，收回原来分配出去的IP地址。接着，LCP释放数据链路层连接。最后释放的是物理层的连接。

使用广播信道的数据链路层

局域网的数据链路层

局域网最主要的特点是：网络为一个单位所拥有，且地理范围和站点数据均有限。
局域网可按网络拓扑进行分类，分为星形网，环形网，总线网。

共享信道要着重考虑的一个问题就是如何使众多用户能够合理方便地共享通信媒体资源。在技术上有两种方法：
（1）静态划分信道，如频分复用，时分复用，波分复用和码分复用。但这种划分信道地方法代价较高，不适合局域网使用。
（2）动态媒体接入控制，它又称为多点接入（multiple access），其特点是信道并非在用户通信时固定分配给用户。这里又分为以下两类：

  • 随机接入 所有的用户都可以随机地发送信息。但如果恰巧有两个或更多的用户在同一时刻发送消息，那么在共享媒体上就要产生碰撞，因此，必须要有解决碰撞的网络协议。
  • 受控接入 用户不能随机地发送信息而必须服从一定地控制。典型代表有分散控制的令牌环局网和集中控制的多点线路探询（轮询）。

适配器的作用

计算机与外界局域网的连接是通过适配器。适配器本来是在主机箱内插入的一块网络接口板。这种接口板又称为网络接口卡NIC（Netwo Interface Card），或简称为网卡。
适配器的重要功能：

  • 进行串行/并行转换
  • 对数据进行缓存
  • 在计算机的操作系统安装设备驱动程序
  • 实现以太网协议

CSMA/CD协议

为了通信的简便，以太网采取了以下两种措施：

  • 第一，采用较为灵活的无连接的工作方式，即不必先建立连接就可以直接发送数据。适配器对发送的数据帧不进行编号，也不要求对方发回确认。以太网提供的服务是尽最大努力的交付，即不可靠的交付。当目的站收到有差错的帧时，就把帧丢弃，其他什么也不做。对有差错帧是否需要重传则由高层来决定。对于重传帧来说，以太网并不知道，而是当作新的数据帧来发送。总线上有一台计算机在发送数据，总线的传输资源就被占用。因此，在同一时间只能允许一台计算机发送数据，否则各计算机之间就会互相干扰，使得所发送的数据被破坏。以太网采用最简单的随机接入。 以太网使用的协议是载波监听多点接入/碰撞检测CSMA/SD（Carrier Sence Multiple Access with Collidion Detection）。
  • 第二，以太网发送的数据都使用曼彻斯特编码（Manchester）。

下面介绍CSMA/CD协议的要点：

  • 多点接入 说明这是总线形的网络。
  • 载波监听 也就是边发送边监听。就是不管在想要发送数据之前，还是发送主句之中，每个站必须不停的检测信道。
  • 碰撞检测 适配器边发送数据边检测信道上的电压的变化情况。当信号电压变化幅度超过一定的门限值时，表明产生了碰撞。

为什么每一个站在发送数据之前已经监听到信道为空闲，还会出现数据在总线上的碰撞呢？这是因为电磁波在总线上的传播速率是有限的。在局域网的分析中，常把总线上的单程端到端传播时延记为τ\tauτ。发送数据的站最迟要经过两倍的总线端到端的传播时延（2τ\tauτ) ，或总线的端到端往返时延。

显然，在使用CSMA/CD协议时，一个站不可能同时进行发送和接收（但必须边发送边监听信道），因此，使用CSMA/CD协议的以太网只能进行双向交替通信（半双工通信）。

由此可见，每一个站在自己发送数据之后的一小段时间内，存在着遭遇碰撞的可能性，这一小段时间是不确定的，因此，以太网不能保证在检测到信道空闲后的某一段时间内，一定能把自己的数据发送出去，这一特点称为发送的不确定性。

最先发送数据帧的A站，在发送数据帧后至多经过2τ\tauτ就可以知道数据帧是否遭受到了碰撞。因此，以太网端到端往返时间2τ\tauτ称为争用期，又称为碰撞窗口。即经过争用期这段时间还没有检测到碰撞，才能肯定这次发送不会发生碰撞。

每一个正在发送数据的站，一旦发现总线上出现了碰撞，就要立刻停止发送，再继续发送若干比特的人为干扰信号，以便让所有用户都知道已经发生了碰撞。然后等待一段随机时间后再次发送。

以太网使用截断二进制指数退避（truncated binary exponential backoff）算法来确定重传的时机：

  • 基本退避时间为争用期2τ\tauτ，具体争用期时间是51.2μ\muμs,对于10Mbit/s以太网，在争用期发送512比特，即64字节。
  • 从整数集合[0,1,...,(2k2^{k}2k-1)]中随机地取出一个数，记为r，重传所需地时延就是r倍地基本退避时间。
  • 参数k按下面地公式计算：
    k=Min[重传次数，10]
  • 当k<=10时，参数k等于重传次数
  • 当重传16次仍不能成功时即丢弃该帧，并向高层汇报。

假如某一个站发送了一个很短地帧，但在发送完毕之前并没有检测出碰撞。假定这个帧再继续向前传播到达目的地之前和别的站发送的帧发生了碰撞，因而目的站将受到有差错的帧，可是发送站却不知道这种情况。因此，以太网规定了一个最短帧长64字节，即512比特。对于10Mbit/s的以太网，发送512比特需要51.2μ\muμs，也就是上面提到的争用期。所以，凡是长度小于64字节的帧都是由于冲突而异常终止的无效帧。只要收到这种无效帧，就应当立即将其丢弃。

以太网还规定了帧间最小间隔为9.6μ\muμs，相当于96比特的时间。这样做是为了使刚刚收到数据帧的站的接收缓存来得及清理，做好接收下一帧的准备。

CSMA/CD协议的工作流程


（1）准备发送：适配器从网络层获得一个分组，加上以太网的首部和尾部，组成以太网帧，放入适配器的缓存中。但在发送之前必须先检测信道。
（2）检测信道：若检测到信道忙，则继续不停地检测，一直等待信道转为空闲。此时若在96比特时间内信道保持空闲（保证了帧最小间隔），就发送这个帧。
（3）在发送过程中仍不停地检测信道，即网络适配器边发送边监听。这里只有两种可能性：

  • 发送成功：如果在争用期内一直未检测到碰撞，就认为发送成功（如果接收方收到了有差错的帧，就丢弃它，后续的工作交给高层来处理）。发送完毕之后，其他什么也不做，回到（1）。
  • 发送失败：在争用期内检测到碰撞。这时立刻停止发送数据，并按规定发送人为干扰信号。适配器接着就执行指数退避算法，等待r倍512比特时间后，返回到步骤（2），继续检测信道。但若16次仍不成功，则停止重传而向上报错。

以太网每发送完一帧，一定要把已发送的帧暂时保留一下。如果在争用期内检测出发生了碰撞，那么还要推迟一段时间后再把这个暂时保留的帧重传一次。

使用集线器的星形拓扑

集线器的一些特点如下：
（1）使用集线器的局域网在物理上是一个星形网，在逻辑上仍是一个总线网，各站共享逻辑上的总线，使用的还是CSMA/CD协议（具体来说，是各站的适配器执行CSMA/CD协议），并且同一时刻至多允许一个站发送数据
（2）集线器很像一个多端口的转发器
（3）集线器工作在物理层，每个端口仅仅简单地转发比特，不进行碰撞检测。
（4）采用了专门地芯片，进行自适应串音回波抵消，减少了近端串音。

以太网的MAC层

MAC层的硬件地址

在局域网中，硬件地址又称为物理地址或MAC地址。802标准中所说的“地址”严格的讲应当是每一个站的“名字”或标识符。“名字指出我们要寻找的那个资源，地址指出那个资源在何处，路由告诉我们如何到达该处”。

IEEE802标准规定MAC地址字段可采用6字节（48位）或2字节（16位）的一种。一个地址块可以生成2242^{24}224个不同的地址。他的通用名称是EUI-48。

  • IEEE的注册管理机构RA负责向产家分配地址字段6个字节的前三个字节（即高24位），称为组织唯一标识符。
  • 地址字段6个字节的后三个字节（即低24位）由产家自行指派，称为拓展唯一标识符。
  • 第一字节的最低有效位为I/G位。表示Individual/Group。当I/G位为0时，地址字段表示一个单个站地址。当I/G位为1时表示组地址，用来进行多播。
  • 第一字节的最低第二位规定为G/L位。表示Global/Local。当G/L位为0是全球管理（保证全球没有相同的地址）。当G/L位为1时是本地管理， 用户可以任意分配网络上的地址。

适配器有过滤功能，当适配器从网络上每收到一个MAC帧就先用硬件检查MAC帧中的目的地址。如果是本站的则收下，否则将其丢弃。发往本站的帧包括以下三种：

  • 单播（unicast）帧 收到的帧MAC地址与本站的MAC地址相同。
  • 广播（broadcast）帧 发送给本局域网上所有站点的帧（全1地址）
  • 多播（multicast）帧 发送给本局域网一部分站点的帧。

MAC帧的格式

常用的以太网MAC帧格式有两种标准：DIX Ethernet V2 标准和IEEE 802.3标准，最常用的MAC帧是以太网V2的格式。

  • 前两个字段分别为6字节长的目的地址和源地址字段。第三个字段是2字节的类型字段，用来标志上一层使用的是什么协议，如0x0800表示上一层使用的是IP数据报。第四个字段是数据字段，其长度在46字节和1500字节之间，当长度不足46字节时，会在数据字段后面加入一个整数字节的填充字段。最后一个字段是4字节的帧检验序列FCS。
  • 在帧前面插入（硬件生成）的8字节中，第一个字段共七个字节，是前同步码，用来迅速实现MAC帧的比特同步。第二个字段1字节是帧开始定界符，表示后面的信息就是MAC帧。

无效的MAC帧（简单地丢弃）：

  • 数据字段的长度与长度字段的值不一致
  • 帧的长度不是整数个字节
  • 用收到的帧检验序列FCS查出有差错
  • 数据字段的长度不在46~1500字节之间

IEEE802.3MAC帧格式：

  • IEEE802.3规定地MAC帧地第三个字段是“长度/类型”
    • 当这个字段值大于0x0600时（相当于十进制的1536），就表示类型。这样的帧和以太网V2MAC帧完全一样。
    • 当这个字段值小于0x0600时才表示长度。数据字段必须装入上面的逻辑链路控制LLC子层的LLC帧。

以太网交换机的特点

  • 以太网交换机实质上就是一个多接口的网桥。
  • 每个接口都直接与一个单台主机或另一个以太网交换机相连，并且一般都工作在全双工方式。
  • 以太网交换机具有并行性。
  • 以太网交换机的接口有存储器，能在输出繁忙时把到来的帧进行缓存。
  • 以太网交换机是一种即插即用设备，其内部的帧交换表（又称为地址表）是通过自学习算法自动地逐渐建立起来的。
  • 以太网交换机使用了专用地交换结构芯片，用硬件转发，其转发速率要比使用软件转发地网桥块很多。
  • 用户独享带宽，增加了总容量。
  • 以太网交换机地交换方式：
    • 存储转发方式，把整个帧先缓存后再处理。
    • 直通方式，接收数据帧地同时立即按数据帧的目的地址决定该帧的转发接口。

以太网交换机的自学习功能

以太网运行自学习算法自动维护交换表。开始时，交换表是空的。




交换机自学习和转发帧的步骤归纳：

  • 交换机收到一帧后先进行自学习。查找交换表与收到帧的源地址有无匹配的项目。
    • 如没有，就在交换表中增加一个项目（源地址，进入的接口和有效时间）
    • 如有，则把原有的项目进行更新（进入的接口或有效时间）
  • 转发帧，查找交换表与收到帧的目的地址有无相匹配的项目。
    • 如没有，则向所有其他接口（除进入的接口外）转发。
    • 如有，则按交换表中给出的接口进行转发。
    • 若交换表中给出的接口就是进入交换机的接口，则应丢弃这个帧。

交换机形成了一个独立的碰撞域，一个更大的广播域。

交换机的兜圈子问题

  • 增加冗余链路时，自学习的过程就可能导致以太网帧在网络的某个环路中无限制地兜圈子。
  • 交换机使用了生成树协议STP（Spanning Tree Protocol），其要点是：不改变网络的实际拓扑，但在逻辑上则切断某些链路，使得从一台主机到所有其他主句的路径是无环路的树状结构，从而消除了兜圈子现象。

从总线以太网到星形以太网

  • 早期，以太网采用无源的总线结构，使用CSMA/CD协议，以半双工方式工作。
  • 现在，采用星形结构，以全双工方式工作，但仍然采用以太网的帧结构。

虚拟局域网

  • 虚拟局域网VLAN是由一些局域网网段构成的与物理位置无光的逻辑组，而这些网段具有某些共同的需求。每一个VLAN的帧都有一个明确的标识符，指明发送这个帧的计算机是属于哪一个VLAN。虚拟局域网其实只是局域网给用户提供的一种服务，而并不是一种新型局域网。
  • 虚拟局域网允许在以太网的帧格式中插入一个4字节的标识符，称为VLAN标记（tag），用来指明发送该帧的计算机属于哪一个局域网。这样的帧称为802.1Q帧或带标记的以太网帧。
